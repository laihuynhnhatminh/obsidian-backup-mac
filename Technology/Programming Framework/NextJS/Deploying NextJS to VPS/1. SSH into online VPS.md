#### Create SSH Key

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
> Choose filename: id_yourinputhere
```

#### Checking SSH public key

```shell
cat ~/.ssh/id_yourinputhere.pub
```

#### Routing SSH Key into config

```powershell
cd ~
cd .ssh
New-Item -ItemType File config
code config
```


```bash
#Default GitHub account
Host github.com
HostName github.com
User git
IdentityFile ~/.ssh/id_yourinputhere
```

#### First Connection to VPS to add SSH

> Note: root could be changed depend on your VPS service, remember to properly check them out

```bash
ssh root@ip_address
```

After your first connection

Check if .ssh exist (if not, you can create that folder), otherwise check if `authorized_keys` file exist or not. If not create them similar to how you create config file with command line. After that follow the instruction below

Creating .ssh folder

```bash
mkdir .ssh
```

```bash
cd ~
cd .ssh
sudo nano ~/.ssh/authorized_keys
```

After that copy the `ssh public key` to the file and then save.

#### CLI for SSH connection instead of manually copy them

```bash
ssh-copy-id -i ~/.ssh/id_rsa.pub root@192.0.2.123
```

```bash
Get-Content $env:USERPROFILE\.ssh\id_rsa.pub | ssh <user>@<hostname> "cat >> .ssh/authorized_keys"
```

CONFIG file

```
Host github.com
Hostname github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_nhatminhwork

Host himemiya
HostName 103.155.161.90
User himemiya
IdentityFile ~/.ssh/id_nhatminhwork
Port 22
```

#### Create a new user instead of using root

Using these, it will force you to create a password for this. Do save it somewhere safe (keyvault or password manager)

```bash
adduser yourUserName
```

Changing to the user

```bash
usermod -aG sudo yourUserName
```
